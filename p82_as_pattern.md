**as-pattern**

Это синтаксическая конструкция, которая позволяет объединить сопоставление с образцом и использование именованного исходного значения.
```haskell
имя@(паттерн)
```
Если `паттерн` успешно сопоставляется, переменная `имя` связывается с тем же значением, которое сопоставлялось, а переменные внутри `паттерна` связываются с его частями.

Пример:
```haskell
module Main (main) where

describe :: [Int] -> String
describe lst@(x:xs) = "голова: " ++ show x
         ++ ", хвост: " ++ show xs
         ++ ", весь список: " ++ show lst
describe []         = "Пустой список"

main :: IO ()
main = do
     putStrLn . describe $ [1,2,3,4,5] -- голова: 1, хвост: [2,3,4,5], весь список: [1,2,3,4,5]
```
Без `as-pattern` пришлось бы либо передавать список дважды или восстанавливать его: `x:xs` вместо `lst`, что неудобно или невозможно при сложных паттернах. Так же важно, что при этом не выполняется копирование данных в `lst` - `имя` и `паттерн` указывают на одну и ту же структуру.
