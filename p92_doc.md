**Оформление документации**

Стандартом документации для haskell является haddock. 
Haddock преобразует комментарии особого формата в красивую HTML или PDF документацию, а также индексировать всё это поисковой системой Hoogle.

Ниже представлен пример с принятыми форматами комментариев:
```haskell
{- |
Модуль для сложения двух чисел.

Этот модуль предоставляет функцию @add@ для сложения двух целых чисел.

Пример использования:

> let sum = add 2 3
> print sum
-}
module Sum (
       -- * Функция сложения
           add
       ) where

-- | Складывает два числа, переданных в качестве аргументов
--
-- >>> add 2 3
-- 5
--
-- >>> add 10 1
-- 11
add :: Int -- ^ первое число
    -> Int -- ^ второе число
    -> Int -- ^ результат сложения
add a b = a + b
```

Перед объявлением модуля следует писать комментарии к модулю. Можно использовать `-- * Заголовок` для создания разделов в экспортируемом списке модуля. Это сильно улучшает навигацию.
В описании функции удобно использовать маркер `-- ^` (каретка), который прикрепляет комментарий к элементу, находящемуся перед ним (в многострочных сигнатурах).
Еще поддерживаются вычисляемые примеры через `-- >>>`, которые сами обновляются при изменении кода функции.

Внутри комментариев haddock поддерживает специальную разметку:
- Ссылки на функции/типы - `'имяФункции'` или `'ИмяТипа'`,
- Ссылки на модули - `"Имя.Модуля"`,
- Моноширинный текст (код) - `@переменная@` или ``переменная``,
- Блоки кода - через `@ ... @` или через "птички" (bird tracks) `> код`,
- Жирный текст: `__жирный__`,
- Курсив: `/курсив/`.

Чтобы документация генерировалась и включала все модули, убедитесь, что в вашем .cabal файле (или package.yaml) включена генерация Haddock:
```haskell
haddock-options:     --hoogle --hyperlink-source
```
где:
- `--hoogle` генерирует текстовый файл для индексации в hoogle,
- `--hyperlink-source` делает имена функций в примерах кода кликабельными ссылками на исходный код.

Затем можно сгенерировать локальной файл документации через `cabal haddock` или `stack haddock` в которые можно добавить аргументы:
- `--haddock-hoogle` - создаст текстовый файл базы данных для Hoogle (чтобы локально искать по своему коду),
- `--haddock-hyperlink-source` - сделает имена функций в примерах кода кликабельными ссылками, ведущими на исходный код.
Или сгенерировать файл документации только для одного выбранного `.hs` файла, чтобы проверить как будет выглядеть документация для него: `cabal haddock app/Sum.hs`
