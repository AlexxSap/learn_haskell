**Трансформеры монад**

Бывает ситуация, когда поведения одной стандартной монады не хватает, например нужно иметь запись чего-то через `Writer` и одновременно чтение чего-то через `Reader`. То есть нужно несколько эффектов в одной функции. Для этого существуют трансформеры монад - они позволяют объединить несколько монад сконструировав из них одну.

Трансформер монад - это конструктор типа, который принимает монаду в качестве аргумента и возвращает монаду как результат. То есть, если монада имеет кайнд `m : * -> *`, то трансформер - `t : (* -> *) -> * -> *`.
Можно объединять несколько трансформеров и формировать из них цепочки: `t1(t2(t3 Identity) a`, но при этом нужно учитывать, что порядок следования тут важен. В качестве основы вложенности монад используются `Identity` или `IO`.

Пример:
```haskell
addInt :: Int -> WriterT [String]
                 (ReaderT Int Identity)
                 Int
addInt a = do
       b <- lift ask
       tell ["add " ++ show a ++ " to " ++ show b]
       pure (a+b)

main :: IO ()
main = print $ runIdentity
             $ runReaderT (
               runWriterT $ addInt 3) 2 -- (5,["add 3 to 2"])
```
Внутри функции `addInt` можно использовать как `ask` (от монады `Reader`), так и `tell` (от монады `Writer`), но с применением специальной функции `lift` (в примере нужно только для `ask`, так как она вложена), которая поднимает вычисление из монады наверх по вложенности в трансформер. 
Если мы используем библиотеку `transformers`, то нам придётся указывать `lift` столько же раз, насколько глубоко лежит нужная монада. 

При использовании `mtl` пример выше можно переписать проще:
```haskell
{-# LANGUAGE FlexibleContexts #-}
-- импорты и прочее пропустим
addInt :: (MonadReader Int m, MonadWriter [String] m) => Int -> m Int
addInt a = do
    b <- ask
    tell ["add " ++ show a ++ " to " ++ show b]
    pure (a + b)

main :: IO ()
main = print $ runIdentity
             $ runReaderT (
               runWriterT $ addInt 3) 2
```
У использования `mtl` есть два ключевых преимущества: не нужно задавать вложенность монад в функции, она определяется при вызове, и не нужно использовать `lift`, библиотека сама сможет разобраться из какой монады вызов (ну, а если не сможет, тогда нужно указывать `lift`'ы).
