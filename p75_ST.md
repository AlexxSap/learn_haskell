**Монада ST**

Это монада для локальных мутабельных вычислений. Она позволяет создавать и изменять ссылки, массивы, хеш-таблицы и другие структуры, но при этом гарантирует, что результат вычисления будет чистым и не будет содержать неподконтрольных побочных эффектов.
В отличии от `State` сабж не копирует данные для изменения, а мутирует их, так что работает значительно быстрее.

```haskell
data ST s a
runST :: (forall s. ST s a) -> a
```
Тип `ST` состоит из значения типа `a` и типовой переменной `s` - она нужна для системы типов, чтобы отделять потоки данных друг от друга.
Так же есть функция распаковки `runST`, которая позволяет достать `a`. 
Квантор `forall s` внутри аргумента означает, что функция должна работать для любого `s`, который выберет вызывающий.

Для работы с `ST` используются:
- тип `STRef s a` - мутабельная ссылка внутри `ST`,
- `newSTRef :: a -> ST s (STRef s a)` - создаёт мутабельную ссылку внутри `ST`,
- `readSTRef :: STRef s a -> ST s a` - прочитать значение,
- `writeSTRef :: STRef s a -> a -> ST s ()` - записать значение,
- `modifySTRef' :: STRef s a -> (a -> a) -> ST s ()` - изменить значение.

Пример:
```haskell
module Main (main) where

import Control.Monad.ST
import Control.Monad
import Data.STRef (newSTRef, modifySTRef, readSTRef)

sumWithST :: Num a => [a] -> a
sumWithST xs = runST $ do
    n <- newSTRef 0
    forM_ xs $ \x -> do 
        modifySTRef n (+x)
    readSTRef n

main :: IO ()
main = do
     print . sumWithST $ [1,2,3,4] 
```
