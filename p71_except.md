**Монада Except**

Описывает вычисление, допускающее возбуждение и перехват исключений. Находится в пакете `transformers` в модуле `Control.Monad.Except`. 
По сути это семантическая обёртка над `Either e a`, для распаковки которого используется функция `runExcept`:
```haskell
newtype Except e a = Except { runExcept :: Either e a }
```
, где `e` - тип ошибки (в `Left`),
`a` - тип успешного результата (в `Right`).
Так же есть функция `except` для упаковки `Either`:
```haskell
except :: Either e a -> Except e a
```

Для работы с `Except` доступны функции:
- `throwE` - немедленно завершает вычисление ошибкой. Как только возникает `throwE`, все последующие `>>=` игнорируются, пока ошибка не будет поймана `catchE`.
```haskell
throwE :: e -> Except e a
```
- `catchE` - ловит ошибку и позволяет обработать или преобразовать её.
```haskell
catchE :: Except e a -> (e -> Except e' a) -> Except e' a
```

Использование:
```haskell
do { action1; action2; action3 } `catchE` handler
```
 - в блоке `do` запускается несколько действий, которые могут возбуждать исключения, которые обрабатываются в `handler`.

Чаще всё же используется просто `Either`, а `Except` - там где нужна семантика обработки ошибок.

Есть еще библиотека `mtl` (monad transformer library), которая содержит более высокоуровневые абстракции для работы с монадами. Она внутри себя использует `transformers`. В библиотеке `mtl` те же функции называются `throwError` и `catchError`.
