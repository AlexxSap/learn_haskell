**Экзистенциальные типы**

Haskell это язык со строгой статической типизацией. Эта строгость не позволяет запросто создавать гетерогенные коллекции (как в LISP или JS) или динамически определять поведение объектов в рантайме (как в С++ и прочем таком). Однако такая возможность есть благодаря экзистенциальным типам. 

Обычно при конструировании типа все типовые переменные встречаются и справа и слева от знака `=`:
```haskell
data Worker b x y = Worker {buffer :: b, input :: x, output :: y}
foo :: Worker [Int] Int Int
```

Но если наш `Worker` может работать с любым типом `b` и мы не хотим требовать его явно в сигнатуре типа, он может определяться автоматически. Для этого существует квантор всеобщности `forall` (он же ∀). Но, чтобы мы могли всё-таки с ним работать, нам нужен какой-то интерфейс - класс типов (в примере ниже это класс типов `Buffer`).
```haskell
data Worker x y = forall b. Buffer b => Worker {buffer :: b, input :: x, output :: y}
foo :: Worker Int Int
```
Для подобного расширения системы типов необходимо включить расширение языка `GADTs` (хотя на GHC2024 у меня работает и без этого расширения). 

Более простой пример:
```haskell
module Main (main) where

data Obj = forall a. (Show a) => Obj a

xs :: [Obj]
xs = [Obj 1, Obj "foo", Obj 'c']

doShow :: [Obj] -> String
doShow [] = ""
doShow ((Obj x):xs) = show x ++ doShow xs

main :: IO ()
main = putStrLn (doShow xs)
```
Здесь мы используем стандартный класс типов `Show` и формируем гетерогенный список `xs` из объектов с разным скрытым (за квантором всеобщности) типом. При этом каждый подлежащий тип имеет свою имплементацию `Show`.
