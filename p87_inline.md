**Встраивание**

`INLINE` - это специальная директива компилятору, которая указывает, что тело функции должно быть подставлено (`inlined``) непосредственно в место её вызова, вместо того чтобы выполнять обычный вызов функции.
```haskell
{-# INLINE add #-}
add :: Int -> Int -> Int
add x y = x + y

main :: IO ()
main = print (add 2 3)
```
Когда компилятор встретит вызов `add 2 3`, он не будет генерировать код для перехода к адресу функции и создания нового стекового кадра. Вместо этого он берет тело функции `(x + y)` и подставляет его прямо в точку вызова, заменяя формальные параметры (x, y) на фактические аргументы (2, 3). В результате выражение `add 2 3` превращается в `2 + 3`, которое оптимизатор затем может свернуть (constant folding) в `5` ещё на этапе компиляции.

Встраивание полезно для оптимизации кода (устранение оверхеда абстракций), свёртки констант и оптимизации работы с коллекциями функциями `map`, `filter` и др.
Однако деректива `INLINE` не гибкая и будет вставлять тело фунцкии где только можно, даже если это сульно раздует код в итоге. Чтобы переложить решение встраивания на компилятор есть директива `INLINABLE`. Она говорит компилятору, что функцию хорошо бы встроить, но окончательное решение за компилятором - будет это хорошо или нет.

Кроме того встраивание может приводить к дублированию вычислений из-за ленивости. Например, если функция вычисляет что-то дорогое и используется дважды, компилятор мог бы вычислить это один раз и сохранить результат (thunk). Но при обязательном инлайнинге компилятор может решить, что дешевле вычислить выражение дважды, чем создавать и проверять thunk, что приведёт к повторным вычислениям.

Чтобы явно избежать встраивания функции, если это гарантировано приводит к проблемам, следует использовать директиву `NOINLINE` - она говорит компилятору, что встраивать эту функцию нельзя.
