**Призмы**

Это расширение линз, которое позволяет работать с типами-суммами в стиле линз заменяя стандартное сопоставление с образцом. Призмы позволяют извлекать значения из конкретного конструктора, конструировать значения и модифицировать их, но с одной важной особенностью: извлечение может завершиться неудачей, поэтому оно всегда возвращает `Maybe`.

Основные операции:
- `preview :: s -> Maybe a` или `(^?)` - извлечь значение,
- `review :: a -> s` или `(#)` - создать значение,
- `over :: (a -> a) -> s -> s` или `(%~)` - изменить значение, если конструктор совпал,
- `set :: a -> s -> s` или `(.~)` - заменить значение, если конструктор совпал,
- `toListOf` или `(^..)` - собрать все совпавшие значения в список,
- `match` или `(^#)` - попытаться извлечь, если не вышло — вернуть исходное значение типа `Either s a`.

Для стандартных типов-сумм уже есть призмы:
- `Maybe` - `_Just`и `_Nothing`,
- `Either` - `_Left` и `_Right`,
- Списки `[]` - `_Cons` (голова и хвост) и `_Nil`.

Пример:
```haskell
-- если у нас есть такой тип
data Town = Amsterdam (Maybe Int)
          | Moscow (Maybe String)
          | London (Maybe String)
     deriving Show
-- создаём для него призму
makePrisms ''Town
...
-- далее где-то создаём переменную
let myTown = Amsterdam (Just 33)
-- и читаем значение
print (myTown ^? _Amsterdam . _Just) -- Just 33
print (myTown ^? _Moscow . _Just)    -- Nothing
```
Чтобы избежать конфликта имён автоматически создаваемые призмы получают имена, которые начинаются с нижнего подчёркивания.

Призмы удобно использовать для фильтрации списков таких значений:
```haskell
let lst = [Amsterdam (Just 1),
           Moscow (Just "one"),
           Amsterdam Nothing,
           London (Just "one")]
print (lst ^.. traverse . _Amsterdam . _Just) -- [1, 33]
print (lst ^.. traverse . _London . _Just)    -- ["one"]
```

Естественно, что призмы легко компонуются с линзами через композицию `(.)`.
