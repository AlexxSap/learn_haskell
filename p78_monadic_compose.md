**Монадическая композиция**

позволяет соединить монадические вычисления в одно большое монадческое вычисление. Монадическая композиция позволяет писать код так, будто все вычисления успешны и чисты, перекладывая на компилятор ответственность за проброс контекста, обработку ошибок и управление эффектами. Благодаря этому код становится более декларативным и компактным.

Для такого соединения есть несколько операторов:
- `>>= :: m a -> (a -> m b) -> m b` - связывание - передаёт результат из монады в следующую функцию - огранизует цепочку действий в которой каждое последующее зависит от результата предыдущего,
- `>> :: m a -> m b -> m b` - выполняет два действия, игнорируя результат первого,
- `>=> :: (a -> m b) -> (b -> m c) -> (a -> m c)` - композиция Клейсли - транзитивное выполнение,
- `<=< :: (b -> m c) -> (a -> m b) -> (a -> m c)` - обратная композиция Клейсли, аналог `(.)` в мире монад - `(f <=< g) x` эквивалентно `g x >>= f`.

Пример без композиции с явными проверками результатов:
```haskell
process :: String -> Maybe Result
process raw = case parse raw of
  Nothing -> Nothing
  Just p  -> case validate p of
    Nothing -> Nothing
    Just v  -> case transform v of
      Nothing -> Nothing
      Just r  -> Just r
```

То же самое, но с использованием композиции:
```haskell
process :: String -> Maybe Result
process = parse >=> validate >=> transform
```
При таком подходе компилятор сам прервёт цепочку вычислений, если где-то случится `Nothing`. Нам остаётся только описать pipeline вычисления. Еще важно то, что компилятор не даст соединить вычисления с разными монадами без явного преобразования (например через lift или трансформеры).
