**Мультипараметрические классы типов**

Стандартно классы типов параметризуются для одного типа и, соответственно, одним типом реализуются:
```haskell
-- а - тип параметр
Class ClassName a where ...
-- класс типа реализуется для Int
instance ClassName Int where ... 
```
Но иногда бывает нужно сделать класс типа для некоторой комбинации типов, которые могут входить в одну операцию. Например перемножение матриц, векторов и чисел.
Такую возможность добавляет расширение `MultiParamTypeClasses`. 

Можно создать класс вида:
```haskell
class Meow a b c where
    (^.^) :: a -> b -> c
```
, который параметризуется тремя типами, а точнее отношением этих типов.
Однако в этом случае придётся явно указывать тип результата через аннотацию типов:
```haskell
instance Meow A B C where ...
...
let a = A {...}
let b = B {...}
let c::C = (a ^.^ b)
```
Это не удобно и для решения этой проблемы есть расширение `FunctionalDependencies`, которое позволяет добавить функциональную зависимость в определение класса. Она определяет, что тип `с` уникальным образом определяется типами `a` и `b`:
```haskell
class Meow a b c | a b -> c where ...
```

Эти расширения не входят в стандарт, хотя и очень широко используются, потому что нет строгой математической теории, которая описывала бы исчерпывающе такие отношения типов, что может приводить к неоднозначности при воводе типов.
