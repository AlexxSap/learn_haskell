**Стандартная монада Either**

Это тип, который представлет значение одного из двух возможных типов - правого или левого. Тип `Either` можно использовать как расширенную версию `Maybe` для явной обработки ошибок, когда возможны несколько неудачных исходов и нам нужно их отличать, или как сутуацию с двумя взаимоисключающими исходами.
```haskell
data Either a b = Left a | Right b
```
Имена для конструкторов `Left` и `Right` произвольны, но исторически так сложилось, что `Left` используют для неудачного исхода, а `Right` - для удачного.

Пример:
```haskell
takeValue :: Int -> Int -> Either String Int
takeValue n all
    | n < 0 = Left "нельзя взять отрицательное количество"
    | n > all = Left "слишком много просишь"
    | otherwise = Right (all - n) 
```

Естественно, что `Either` можно использовть в монадических вычислениях:
```haskell
calc :: Either String Int
calc = do
     a <- takeValue 3 10
     b <- takeValue 11 3
     pure (a+b)
```

Для типа `Either` есть функция `either`, которая позволяет обработать значения с `Left` и `Right` без использования сопоставления с образцом:
```haskell
either :: (a -> c) -> (b -> c) -> Either a b -> c
```
тут `(a -> c)` - это обработчик для `Left`, а `(b -> c)` - для `Right`.

```haskell
showResult :: Either String Int -> String
showResult = either (\err -> "Ошибка: " ++ err) 
                    (\val -> "Теперь осталось: " ++ show val)
```
В качестве функции-обработчика `Left` значения можно указать логирование, сообщение об ошибке, откат каких-то изменений и всякое такое.
